# Схема кодирования текста на кириллице

Схема использует **переменную длину кодов** (prefix-free дерево): самые частые символы получают короткие коды (3 бита), редкие — длиннее. Для редких букв, цифр, регистра и пунктуации применяется **escape-механизм через `111`**.

---

## Принцип декодирования

Декодер читает биты по одному и определяет символ по первым битам:

| Начало кода | Длина   | Что это              |
| ----------- | ------- | -------------------- |
| `010`–`110` | 3 бита  | символ готов         |
| `000`, `001`| 5–7 бит | внутренние узлы, читать дальше |
| `111`       | 7–13 бит| ESC: читать ещё 4 бита для категории |

> `000` и `001` — не символы, а **префиксы** более длинных кодов (внутренние узлы дерева).

---

## Частотное обоснование

Чем чаще символ встречается в русском тексте, тем короче его код:

| Символ  | Частота | Длина кода |
| ------- | ------- | ---------- |
| пробел  | ~17%    | 3 бита     |
| о       | ~11%    | 3 бита     |
| е       | ~8.4%   | 3 бита     |
| а       | ~8%     | 3 бита     |
| и       | ~7.4%   | 3 бита     |
| н, т, с | ~5–7%   | 5 бит      |
| р, в, л | ~4–5%   | 6 бит      |
| к, м, д, п, у, я, ы, ь, г, б | ~1–3.5% | 7 бит |
| запятая | ~0.5%   | 5 бит      |

## 5. ESC-расширение (`111` + 4 бита = 16 слотов)

| Код        | Значение                    |
| ---------- | --------------------------- |
| `111 0000` | з                           |
| `111 0001` | ч                           |
| `111 0010` | й                           |
| `111 0011` | х                           |
| `111 0100` | ж                           |
| `111 0101` | ш                           |
| `111 0110` | ю                           |
| `111 0111` | э                           |
| `111 1000` | щ                           |
| `111 1001` | ф                           |
| `111 1010` | ц                           |
| `111 1011` | ё                           |
| `111 1100` | ъ                           |
| `111 1101` | → Цифры                     |
| `111 1110` | → CAPS (регистр)            |
| `111 1111` | → Пунктуация / Управляющие  |

---

## 6. Цифры (`111 1101` + 2 бита режима)

| Код           | Значение                                  |
| ------------- | ----------------------------------------- |
| `111 1101 00` | Одиночная цифра + 4 бита                  |
| `111 1101 01` | Серия цифр: по 4 бита + `1111` (стоп)     |
| `111 1101 10` | зарезервировано                           |
| `111 1101 11` | зарезервировано                           |

**Таблица цифр (4 бита):**

| Цифра | Код    | Цифра | Код    |
| ----- | ------ | ----- | ------ |
| 0     | `0000` | 5     | `0101` |
| 1     | `0001` | 6     | `0110` |
| 2     | `0010` | 7     | `0111` |
| 3     | `0011` | 8     | `1000` |
| 4     | `0100` | 9     | `1001` |
| стоп  | `1111` |       |        |

Пример — число `42`: `111 1101 00 0100` (одиночная цифра 4), затем `111 1101 00 0010` (одиночная цифра 2).
Серия `2025`: `111 1101 01` + `0010 0000 0010 0101` + `1111`.

---

## 7. Регистр (`111 1110` + 2 бита)

Правило: CAPS-код всегда идёт **перед** кодом буквы.

| Код           | Значение                                 |
| ------------- | ---------------------------------------- |
| `111 1110 00` | CAPS_NEXT — следующий символ заглавный   |
| `111 1110 01` | CAPS_ON — все следующие буквы заглавные  |
| `111 1110 10` | CAPS_OFF — выключить режим заглавных     |
| `111 1110 11` | зарезервировано                          |

---

## 8. Пунктуация и управляющие (`111 1111` + 4 бита)

| Код            | Символ        |
| -------------- | ------------- |
| `111 1111 0000`| `\t` (таб)    |
| `111 1111 0001`| `\n` (новая строка) |
| `111 1111 0010`| EOF (конец потока) |
| `111 1111 0011`| `.` (точка)   |
| `111 1111 0100`| `!`           |
| `111 1111 0101`| `?`           |
| `111 1111 0110`| `:`           |
| `111 1111 0111`| `;`           |
| `111 1111 1000`| `-`           |
| `111 1111 1001`| `(`           |
| `111 1111 1010`| `)`           |
| `111 1111 1011`| `"`           |
| `111 1111 1100`| `«`           |
| `111 1111 1101`| `»`           |
| `111 1111 1110`| `…`           |
| `111 1111 1111`| `№`           |

> **EOF** обязателен: биты хранятся в байтах, и без маркера конца декодер прочитает лишние биты-заполнители как символы.
